## How to use it
To start the API server locally on your machine, you can run this command in your terminal:
```bash
uvicorn api:app --reload
```
Once it's running, you can access the interactive API documentation automatically generated by FastAPI at [http://127.0.0.1:8000/docs](http://127.0.0.1:8000/docs). This interactive page lets you easily test the endpoint directly from your browser!

## Batch Analytics & Duplication Prevention (Update)
The application has been modified to properly handle identical/duplicate reviews without triggering memory leaks or cross-client contamination.

### 1. `/analyze-batch`
You can now submit an entire batch of reviews at once using the new `POST /analyze-batch` endpoint.
```json
{
  "reviews": [
    {"review": "This is great!", "rating": 5},
    {"review": "This is great!", "rating": 5}
  ]
}
```
**Benefits:**
- **Incredibly Fast**: It analyzes all reviews concurrently over a single HTTP request network bounce.
- **Isolated Duplications**: The second "This is great!" review naturally triggered a mathematically heavy penalty (50-points), but only within the context of this specific array—meaning a different batch isn't punished for similar wording.

### 2. Session Context for `/analyze`
The baseline `POST /analyze` structure still exists for single-review ingestion. If you want to group these single requests over time to catch slow-drip duplicates, you can now pass an optional `client_id` standard string in your JSON body:
```json
{
  "client_id": "session_user_456",
  "review": "Nice product.",
  "rating": 5
}
```
*Note: Any requests without a explicitly provided `client_id` will implicitly group together under a default key.*
